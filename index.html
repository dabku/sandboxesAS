🔴 Attack Demos

Demo 1 Attack

Bare Agent with Bash

A simple agent with unrestricted bash access. Works great for happy-path analysis — but sets the stage for what's next.

 Demo 2 Attack

Prompt Injection & File Leak

The same agent leaks .env secrets — triggered by a hidden instruction inside a CSV file, or via a direct user request.

 Demo 3 Attack

Naive Docker — Env Var Leak

Agent runs inside Docker (isolated filesystem) but still leaks all environment variables: DATABASE_URL, API keys, and more.

 Demo 4 Attack

Shared Container — Tenant Leak

Two agents share one Docker container. Agent B (Company B) reads Agent A's confidential financial report from the shared filesystem.

🟢 Defense Demos

Demo 5 Defense

Docker Per-Run Sandbox

Fresh container per execution — no secrets passed, read-only data mount, auto-destroyed after use. Much better, but with container overhead.

 Demo 6 Defense

Deno Sandbox

Deno's deny-by-default permission model: only ./sample-data/ is readable. No network, no env vars, no subprocesses — enforced at runtime.

 Demo 7 Defense

Daytona Cloud Sandbox

Production-grade cloud sandbox: fully isolated environment, own filesystem, snapshot-based fast provisioning, auto-cleanup. Simple SDK.
